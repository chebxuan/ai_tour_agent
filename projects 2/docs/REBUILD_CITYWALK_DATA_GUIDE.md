# 城市漫步数据重构工具使用说明

## 📋 功能概述

`rebuild_citywalk_data.py` 是一个数据重构脚本，用于将上海城市叙事数据按故事线重组并优化。

### 核心功能

1. **故事线映射**：根据区域（region）自动映射到对应的故事线（storyline）
2. **节点排序优化**：按重要性（sequence_weight）降序排列，避免连续视觉节点
3. **数据补全**：自动转换数据类型，生成免费标识字段
4. **统计报告**：生成详细的节点分布和时长统计

---

## 🚀 使用方法

### 1. 运行脚本

```bash
# 激活虚拟环境
source venv/bin/activate

# 运行重构脚本
python "projects 2/scripts/rebuild_citywalk_data.py"
```

### 2. 输入输出

**输入文件**：
- `projects 2/city_narratives.csv` - 原始城市叙事数据（165个节点）

**输出文件**：
- `data/citywalk/narratives_v2.csv` - 重构后的数据（包含7个字段）

## 🔧 数据处理逻辑

### 1. 节点排序优化

**第一步：按权重降序**
- 根据 `sequence_weight` 字段降序排列
- 权重越高，节点越重要，排名越靠前

**第二步：节奏过滤**
- 检测连续3个"视觉"类型节点
- 在中间插入"味觉"（咖啡店）或"放空"（休息）节点
- 避免游览节奏单调

### 2. 数据补全

**dwell_time_min 转换**：
```python
# 原始值可能是字符串或小数
"20" → 20
"30.5" → 30
"" → 20 (默认值)
```

**is_free 自动生成**：
```python
# 检测 practical_info 中的关键词
"免费开放" → is_free=True
"无门票信息" → is_free=True
"门票20元" → is_free=False
```

## 🎯 应用场景

1. **Citywalk 路线规划**：按故事线组织游览节点
2. **费用计算**：通过 `is_free` 字段快速筛选免费/付费景点
3. **节奏优化**：视觉、味觉、文化节点交替，提升游览体验
4. **数据分析**：统计各故事线的节点分布和时长

## ⚙️ 自定义配置

### 修改故事线映射

```python

```

### 修改默认故事线

```python
DEFAULT_STORYLINE = "城市经典：B面上海"  # 或其他名称
```

### 调整节奏过滤规则

在 `optimize_node_ordering()` 函数中修改：

```python
# 当前：连续3个视觉节点触发过滤
if consecutive_visual_count == 3:
    # 寻找味觉或放空节点插入

# 可改为：连续2个或4个
if consecutive_visual_count == 2:  # 更严格
if consecutive_visual_count == 4:  # 更宽松
```

## 🐛 故障排查

### 问题1：找不到输入文件

**错误信息**：
```
❌ 错误：找不到输入文件 ...
```

**解决方案**：
- 确认 `projects 2/city_narratives.csv` 存在
- 检查文件路径是否正确

### 问题2：输出目录不存在

**错误信息**：
```
FileNotFoundError: [Errno 2] No such file or directory: ...
```

**解决方案**：
- 脚本会自动创建 `data/citywalk/` 目录
- 确认有写入权限

### 问题3：数据格式异常

**症状**：`dwell_time_min` 转换失败

**解决方案**：
- 脚本会自动使用默认值 20 分钟
- 检查原始 CSV 是否有异常数据
