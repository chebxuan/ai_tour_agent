# 城市叙事引擎 (Urban Narrative Engine) 设计文档

## 核心理念对比

| 维度 | 现有系统：目的地旅游 (Destination-based) | 新模块：城市叙事 (Experience-based) |
|------|----------------------------------------|-----------------------------------|
| **基本单元** | 景点 (POI) | 时刻 (Moment) + 流动 (Flow) |
| **产品形态** | 多日行程套餐 | 半日故事线体验 |
| **用户目标** | 打卡必游景点 | 沉浸城市故事 |
| **时间粒度** | 天 (Day) | 分钟 (Moment) |
| **空间逻辑** | 城市间跳转 | 街区步行流动 |
| **交付物** | 行程单 + 费用表 | 探索地图 (Discovery Map) |

---

## 数据模型：city_narratives.csv

### 字段定义

| 字段名 | 类型 | 示例 | 设计意图 |
|--------|------|------|----------|
| `node_id` | String | `SH-WK-01` | 唯一标识符，程序调用 |
| `city` | String | `上海` | 所属城市 |
| `storyline` | String | `武康路的老洋房故事` | **【核心】** 故事线，筛选和聚合基础 |
| `node_name` | String | `武康大楼的最佳拍摄机位` | 节点具体名称 |
| `node_type` | Enum | `视觉打卡` | **【核心】** 体验类型，用于节奏韵律 |
| `description` | Text | `在淮海中路和武康路交叉口...` | 给用户的"任务卡" |
| `lat_long` | String | `31.20, 121.45` | **【核心】** 经纬度，步行友好计算 |
| `dwell_time_min` | Int | `10` | 建议停留时间(分钟) |
| `serendipity_tip` | Text | `旁边的冰淇淋店有开心果口味...` | **【核心】** 计划外惊喜提示 |
| `host_story` | Text | `据说设计师在这里致敬了一艘轮船...` | 地陪/主持人的"剧本" |

### node_type 类型定义

| 类型 | 说明 | 节奏作用 |
|------|------|----------|
| `视觉打卡` | 拍照出片点 | 高潮节点候选 |
| `味觉停留` | 咖啡馆/小吃/餐厅 | 节奏缓冲 |
| `历史锚点` | 有故事的建筑/遗址 | 叙事支撑 |
| `互动体验` | 可参与的手工/活动 | 沉浸加深 |
| `购物发现` | 设计师店/市集/书店 | 节奏调节 |
| `过渡节点` | 街角/弄堂/桥 | 流动连接 |

## 引擎算法逻辑

### 输入
- `storyline`: 用户选择的故事线
- `duration_min`: 期望时长（分钟）

### 算法步骤

```
1. 筛选 (Filter)
   └── 从 city_narratives.csv 筛选 storyline 匹配的节点

2. 聚类 (Cluster)
   └── 基于 lat_long 进行地理位置聚类
   └── 剔除离群点，确保步行友好（节点间 < 500m）

3. 排序与编排 (Sequence & Pace)
   ├── 节奏韵律原则：避免连续相同 node_type
   ├── 高潮节点识别：视觉打卡/历史锚点 → 安排在中后段
   ├── 起点选择：交通便利的过渡节点
   └── 终点选择：有收尾感的味觉停留或视觉打卡

4. 生成 (Generate)
   └── 输出：探索地图 (Discovery Map) JSON
```

## 与现有系统的关系

```
┌─────────────────────────────────────────────────────────┐
│                    Hexa Blueprint™                      │
├─────────────────────┬───────────────────────────────────┤
│  目的地旅游引擎      │      城市叙事引擎                  │
│  (Destination)      │      (Experience)                 │
├─────────────────────┼───────────────────────────────────┤
│  Product Logic      │  city_narratives.csv              │
│  Library_final_     │  - storyline                      │
│  updated.csv        │  - node_type                      │
│  - 多日行程套餐      │  - lat_long                       │
│  - 费用计算         │  - serendipity_tip                │
├─────────────────────┼───────────────────────────────────┤
│  api_main.py        │  api_main.py                      │
│  - /api/v1/complete │  - /api/v1/citywalk (新增)        │
│  - /api/v1/cost     │                                   │
├─────────────────────┼───────────────────────────────────┤
│  扣子 Bot: 行程规划  │  扣子 Bot: 城市漫步 (可复用Bot)    │
└─────────────────────┴───────────────────────────────────┘
```

**设计原则：完全独立，互不干扰**
- 两个引擎可以共存于同一个 API 服务
- 用户可以在同一个 Bot 中选择「规划行程」或「城市漫步」
- 数据文件完全分离，避免字段冲突
