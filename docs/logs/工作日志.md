#### 第二阶段：引擎原型开发 - 完成

**创建** `narrative_engine.py`

核心功能模块：
| 函数 | 职责 |
|------|------|
| `haversine_distance()` | 计算两点间地理距离（米） |
| `load_nodes()` | 从 CSV 加载节点数据 |
| `filter_by_storyline()` | 按故事线筛选节点 |
| `cluster_by_proximity()` | 地理位置聚类，剔除离群点（默认500m阈值） |
| `calculate_rhythm_score()` | 计算序列节奏分数（避免连续相同类型、高潮节点位置优化） |
| `sequence_nodes()` | 排序与编排（最近邻贪心 + 局部搜索优化） |
| `generate_route_hint()` | 生成步行路线提示 |
| `generate_discovery_map()` | 主入口：生成探索地图 JSON |

**算法特点：**
- 最近邻贪心构建初始路径
- 局部搜索优化节奏（交换相邻节点）
- 高潮节点（视觉打卡/历史锚点）优先安排在中后段
- 自动根据时长限制截断节点
- 生成步行距离、时间估算和路线提示

**测试验证：**
- 输入：`武康路的老洋房故事`, 120分钟
- 输出：9个节点，总时长122分钟，总步行距离1449米
- 节奏模式：`视觉→过渡→历史→历史→视觉→购物→味觉→视觉→过渡`
- Flow Score: 1.0

**待优化项：**
- 当前数据只有2个历史锚点且相邻，导致连续相同类型
- 建议增加更多节点类型多样性以优化节奏
